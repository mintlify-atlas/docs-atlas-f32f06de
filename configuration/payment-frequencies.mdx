## Overview

The `PAYMENT_FREQUENCIES` array defines the available billing cycles for your pricing tiers. Users can toggle between these frequencies to see different pricing options.

## Configuration

Define payment frequencies in `config/index.ts`:

```typescript config/index.ts
export const PAYMENT_FREQUENCIES = ["monthly", "yearly"];
```

### Default Setup

The default configuration includes two standard billing cycles:

- **monthly**: Monthly billing
- **yearly**: Annual billing (typically discounted)

These values must match the keys in each tier's `price` object.

## How It Works

The payment frequencies work in conjunction with the tier pricing:

```typescript config/index.ts
export const PAYMENT_FREQUENCIES = ["monthly", "yearly"];

export const TIERS: PricingTier[] = [
  {
    id: "pro",
    name: "Pro",
    price: {
      monthly: 29,  // Shown when "monthly" is selected
      yearly: 24,   // Shown when "yearly" is selected
    },
    // ... other fields
  },
];
```

When users toggle between frequencies, the pricing component displays the corresponding price from each tier's `price` object.

## Customization Examples

### Lifetime Pricing Option

Add a one-time payment option:

```typescript config/index.ts
export const PAYMENT_FREQUENCIES = ["monthly", "yearly", "lifetime"];

export const TIERS: PricingTier[] = [
  {
    id: "pro",
    name: "Pro",
    price: {
      monthly: 29,
      yearly: 24,
      lifetime: 499,  // One-time payment
    },
    description: "For professional developers",
    features: ["Unlimited projects", "Priority support"],
    cta: "Get started",
  },
];
```

### Quarterly Billing

Add quarterly payment cycles:

```typescript config/index.ts
export const PAYMENT_FREQUENCIES = ["monthly", "quarterly", "yearly"];

export const TIERS: PricingTier[] = [
  {
    id: "business",
    name: "Business",
    price: {
      monthly: 99,
      quarterly: 89,   // 10% discount
      yearly: 79,      // 20% discount
    },
    description: "For growing teams",
    features: ["Advanced features", "Team collaboration"],
    cta: "Start trial",
  },
];
```

### Single Frequency (No Toggle)

If you only offer one billing cycle:

```typescript config/index.ts
export const PAYMENT_FREQUENCIES = ["monthly"];

export const TIERS: PricingTier[] = [
  {
    id: "standard",
    name: "Standard",
    price: {
      monthly: 49,  // Only option
    },
    description: "Simple monthly billing",
    features: ["All features included"],
    cta: "Subscribe",
  },
];
```

<Note>
With a single frequency, the toggle will not be displayed in the UI.
</Note>

### Custom Frequency Labels

While the keys are used internally, you can customize the display labels by modifying the pricing component. The default implementation capitalizes the frequency names automatically:

- `monthly` → "Monthly"
- `yearly` → "Yearly"
- `lifetime` → "Lifetime"

Encourage annual subscriptions with significant savings:

## Price Display Logic

The pricing component automatically handles different price types:

### Numeric Prices

When the price is a number, it's formatted as currency:

```typescript
price: {
  monthly: 29,  // Displays as "$29" with "Per month/user" label
}
```

### String Prices

When the price is a string, it's displayed as-is without currency formatting:

```typescript
price: {
  monthly: "Free",     // Displays exactly as "Free"
  yearly: "Custom",    // Displays exactly as "Custom"
}
```

This is defined in `components/PricingCard.tsx`:

```tsx components/PricingCard.tsx
{typeof price === "number" ? (
  <>
    <NumberFlow
      format={{
        style: "currency",
        currency: "USD",
        trailingZeroDisplay: "stripIfInteger",
      }}
      value={price}
      className="text-4xl font-medium"
    />
    <p className="-mt-2 text-xs font-medium">Per month/user</p>
  </>
) : (
  <h1 className="text-4xl font-medium">{price}</h1>
)}
```

## Validation

<Warning>
Ensure every tier's `price` object contains all frequencies defined in `PAYMENT_FREQUENCIES`. Missing keys will cause runtime errors.
</Warning>

### Valid Configuration

```typescript
export const PAYMENT_FREQUENCIES = ["monthly", "yearly"];

export const TIERS: PricingTier[] = [
  {
    id: "pro",
    name: "Pro",
    price: {
      monthly: 29,  // ✓ Both frequencies defined
      yearly: 24,
    },
    // ...
  },
];
```

### Invalid Configuration

```typescript
export const PAYMENT_FREQUENCIES = ["monthly", "yearly"];

export const TIERS: PricingTier[] = [
  {
    id: "pro",
    name: "Pro",
    price: {
      monthly: 29,  // ✗ Missing "yearly" key
    },
    // ...
  },
];
```

## TypeScript Type Safety

The `PricingTier` interface uses a flexible `Record` type for prices:

```typescript config/index.ts
export interface PricingTier {
  price: Record<string, number | string>;
  // ... other fields
}
```

This allows:
- Any string key (matching your payment frequencies)
- Values as either numbers (for currency formatting) or strings (for custom text)
