## Overview

The `TIERS` configuration array defines all pricing plans displayed in your pricing page. Each tier is defined using the `PricingTier` interface and supports flexible pricing models, feature lists, and visual customization.

## PricingTier Interface

The `PricingTier` interface defines the structure for each pricing tier:

```typescript config/index.ts
export interface PricingTier {
  name: string;
  id: string;
  price: Record<string, number | string>;
  description: string;
  features: string[];
  cta: string;
  highlighted?: boolean;
  popular?: boolean;
}
```

### Field Definitions

<ResponseField name="name" type="string" required>
  The display name of the pricing tier (e.g., "Teams", "Enterprise").
</ResponseField>

<ResponseField name="id" type="string" required>
  A unique identifier for the tier, used for tracking and analytics. Use lowercase with hyphens (e.g., "teams", "enterprise").
</ResponseField>

<ResponseField name="price" type="Record<string, number | string>" required>
  An object mapping payment frequencies to prices. Keys should match your `PAYMENT_FREQUENCIES` array. Values can be:
  - **Number**: Numeric price (e.g., `90` for $90)
  - **String**: Custom text (e.g., `"Free"`, `"Custom"`, `"Contact Us"`)
  
  Example:
  ```typescript
  price: {
    monthly: 90,
    yearly: 75
  }
  ```
</ResponseField>

<ResponseField name="description" type="string" required>
  A brief description of the tier, displayed below the tier name (e.g., "Great for small businesses").
</ResponseField>

<ResponseField name="features" type="string[]" required>
  An array of feature strings displayed as a checklist. Each feature is shown with a checkmark icon.
  
  Example:
  ```typescript
  features: [
    "Unlimited phone calls",
    "30 second checks",
    "Up to 6 seats"
  ]
  ```
</ResponseField>

<ResponseField name="cta" type="string" required>
  The call-to-action button text (e.g., "Get started", "Contact Us").
</ResponseField>

<ResponseField name="highlighted" type="boolean" default="false">
  When `true`, the tier card is displayed with inverted colors (dark background) and a decorative grid pattern. Typically used for premium or recommended plans.
</ResponseField>

<ResponseField name="popular" type="boolean" default="false">
  When `true`, displays a "Most Popular" badge and adds a purple outline to the card. The card also features a subtle radial gradient background.
</ResponseField>

## Visual Flags

### Highlighted Flag

The `highlighted` flag transforms a tier card visually:

- Inverted color scheme (dark background, light text)
- Decorative grid pattern overlay
- Used to emphasize premium or recommended plans

```typescript
{
  id: "premium",
  name: "Premium",
  highlighted: true,  // Applies dark theme to this card
  // ... other fields
}
```

### Popular Flag

The `popular` flag adds visual indicators:

- "🔥 Most Popular" badge in the header
- Purple outline around the card
- Radial gradient background effect

```typescript
{
  id: "pro",
  name: "Pro",
  popular: true,  // Shows "Most Popular" badge
  // ... other fields
}
```

<Note>
You can combine both `highlighted` and `popular` flags on the same tier, though this is uncommon in practice.
</Note>

## Best Practices

<AccordionGroup>
  <Accordion title="Keep tier count reasonable">
    Limit tiers to 3-4 options. Too many choices can overwhelm users and reduce conversions.
  </Accordion>
  
  <Accordion title="Use consistent feature formatting">
    Start features with action verbs or quantities. Keep descriptions concise and scannable.
  </Accordion>
  
  <Accordion title="Price ordering matters">
    Define tiers in ascending price order (free → basic → premium → enterprise) for better UX.
  </Accordion>
  
  <Accordion title="Strategic use of flags">
    Use `popular` for your most common choice and `highlighted` for your highest-value offer.
  </Accordion>
  
  <Accordion title="Clear CTAs">
    Use specific call-to-action text. "Get started" for self-service, "Contact sales" for enterprise.
  </Accordion>
</AccordionGroup>
