## Prerequisites

Before installing, ensure you have:

- **Node.js 18+** installed on your system
- A **Next.js 15+** project with **App Router** enabled
- **React 19+** installed
- **Tailwind CSS** configured in your project

<Note>
  This guide assumes you have an existing Next.js project. If you're starting from scratch, create a new Next.js app first:
  ```bash
  npx create-next-app@latest my-app
  ```
</Note>

## Installation Steps

<Steps>
  <Step title="Install Dependencies">
    Install all required packages for the pricing component:

    <CodeGroup>
      ```bash npm
      npm install @number-flow/react motion next-themes lucide-react
      npm install @radix-ui/react-separator @radix-ui/react-slot @radix-ui/react-tooltip
      npm install class-variance-authority clsx tailwind-merge tailwindcss-animate
      ```

      ```bash yarn
      yarn add @number-flow/react motion next-themes lucide-react
      yarn add @radix-ui/react-separator @radix-ui/react-slot @radix-ui/react-tooltip
      yarn add class-variance-authority clsx tailwind-merge tailwindcss-animate
      ```

      ```bash pnpm
      pnpm add @number-flow/react motion next-themes lucide-react
      pnpm add @radix-ui/react-separator @radix-ui/react-slot @radix-ui/react-tooltip
      pnpm add class-variance-authority clsx tailwind-merge tailwindcss-animate
      ```
    </CodeGroup>

    <Tip>
      **Key dependencies explained:**
      - `@number-flow/react` - Animated number transitions for prices
      - `motion` - Animation library (Framer Motion)
      - `next-themes` - Theme switching support
      - `lucide-react` - Icon library
      - Radix UI packages - Accessible component primitives
    </Tip>
  </Step>

  <Step title="Create Utility Functions">
    Create a `lib/utils.ts` file for the `cn` utility function:

    ```typescript lib/utils.ts
    import { clsx, type ClassValue } from "clsx";
    import { twMerge } from "tailwind-merge";

    export function cn(...inputs: ClassValue[]) {
      return twMerge(clsx(inputs));
    }
    ```

    This utility combines Tailwind classes intelligently, preventing conflicts.
  </Step>

  <Step title="Configure Path Aliases">
    Update your `tsconfig.json` to include path aliases:

    ```json tsconfig.json
    {
      "compilerOptions": {
        "paths": {
          "@/*": ["./*"],
          "@/components/*": ["./components/*"],
          "@/lib/*": ["./lib/*"],
          "@/ui/*": ["./components/ui/*"],
          "@/config/*": ["./config/*"]
        }
      }
    }
    ```
  </Step>

  <Step title="Set Up Theme Provider">
    Wrap your app with the `ThemeProvider` in your root layout:

    ```tsx app/layout.tsx
    import { ThemeProvider } from "next-themes";
    import "./globals.css";

    export default function RootLayout({
      children,
    }: {
      children: React.ReactNode;
    }) {
      return (
        <html lang="en" suppressHydrationWarning>
          <body>
            <ThemeProvider
              attribute="class"
              defaultTheme="system"
              disableTransitionOnChange
            >
              {children}
            </ThemeProvider>
          </body>
        </html>
      );
    }
    ```

    <Note>
      The `suppressHydrationWarning` prop is important to prevent hydration warnings when using theme switching.
    </Note>
  </Step>
</Steps>

## Verification

Verify your installation by checking:

<CodeGroup>
  ```bash Check installed packages
  npm list @number-flow/react motion next-themes
  ```

  ```bash Run development server
  npm run dev
  ```
</CodeGroup>

You should see no errors in the console. If you encounter issues:

- Ensure all peer dependencies are satisfied
- Clear your `.next` cache: `rm -rf .next`
- Restart your development server
